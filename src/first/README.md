# 2차원 충돌 시뮬레이션

**일자:** 2026.07.07  
**작성자:** 1124 이현우  
**소스코드 링크:** <https://github.com/Name-hw/Science-Project/tree/main/src/first>  
**시뮬레이션 링크:** <https://name-hw.github.io/Science-Project/src/first/>

## 1. 서론

통합과학 과정에서 학습한 역학 시스템과 충돌 현상을 수학적 공식으로만 이해하는 것을 넘어, 실시간으로 상호작용하는 시뮬레이션으로 직접 구현해 보고자 함. 이를 통해 물리 법칙이 소프트웨어 알고리즘으로 어떻게 구현되는지 탐구하는 것을 목적으로 함.

## 2. 핵심 기능 구현

이 시뮬레이션은 타입스크립트 기반으로, 객체 지향 프로그래밍을 적용하여 물리적 상호작용을 처리함.

1. **물리 객체 추상화:** 질량, 속도, 위치를 갖는 Object 클래스를 설계하여 각 물체의 물리 계산을 처리함.

2. **사용자 상호작용 및 관성:** 마우스/펜 포인터 이벤트를 통해 객체에 외력을 가할 수 있으며, 입력이 종료된 후에는 관성의 법칙에 따라 운동을 유지함.

3. **충돌 감지 및 충격량 연산:** 두 물체의 좌표와 반지름을 바탕으로 충돌을 감지하고, 상태 변화를 처리함.

4. **델타 타임($\Delta t$) 기반 업데이트:** 매 프레임마다 물리 연산을 수행하기 때문에, 모니터 주사율에 독립적인 물리 연산을 위해, 매 프레임마다 시간 변화량을 측정하여 계산에 이용함.

## 3. 구현 방법

프로그램은 화면 렌더링 영역(index.html)과 물리/이벤트 제어 영역(main.ts)으로 분리되어 있으며, 시뮬레이션의 생명 주기는 다음과 같은 흐름을 따름. 각 물체는 반지름의 길이가 같은 원으로 가정함. 두 물체를 각각 물체1, 물체2라고 함. 물체1을 움직여 물체2에 충돌시키고, 물체1이 반드시 정지하는 상황을 가정함.

### 3-1. 초기화

canvas 컨텍스트를 구성하고, 고유 ID와 질량(Mass)이 부여된 두 개의 Object 인스턴스를 메모리에 할당함.

### 3-2. 메인루프

매 프레임 반복 호출됨

- **물리 연산:** 각 물체의 합력을 초기화함. 사용자의 드래그 입력을 추적하여 위치를 보정하고, 물체끼리의 충돌을 감지하여, 두 물체의 현재 위치를 계산함.
- **충돌 처리:** 두 물체의 중심 사이의 거리가 반지름의 합 이하가 되는 순간 충돌 상태로 판정함. 이때 드래그 상태를 해제하고, 물체2에 작용한 충격량을 계산하여 힘을 구하고, 그 힘을 바탕으로 속도를 변화시킴.
- **렌더링:** canvas를 비우고, 현재 계산된 위치에 맞추어 식별 번호를 포함해 물체를 다시 그림.
- **UI 동기화** 초당 프레임(60Hz) 기준의 내부 데이터를 현실 단위계로 변환하여 소수점 둘째 자리만 화면에 표시함.

### 3-3. 리셋

다음 프레임에서 메인루프가 실행되지 않게 하고, 변수들을 초기화한 뒤 위의 과정을 다시 실행함.

## 4. 어려웠던 점

1. **물리 법칙의 알고리즘화:** 학습한 물리 공식을 코드로 변환하기 위해 Object 클래스 내부에 어떤 맴버 변수(속도, 힘 등)와 함수를 배치할지에 대한 설계에 어려움이 있었음.

2. **생명 주기 설계:** _초기화-메인루프-리셋_ 으로 이루어진 생명 주기를 설계하는데에, 메인 루프에서 물리 연산, 렌더링, 입력 처리를 모두 처리하는 구조를 설계하는 것에 많은 시행착오가 있었음.

3. **웹 프론트엔드 및 그래픽 환경 구축:** TypeScript의 타입 시스템과 HTML5 Canvas API를 처음 학습하고 적용하는 과정에서 시행착오를 겪음. 특히 화면에 각 물체의 정보들을 표시하고 canvas를 새로 그리는 루프를 만드는 설계에 고민이 깊었음.

4. **CI/CD 및 호스팅 도입:** 시뮬레이션의 접근성을 높이기 위해 단순 로컬 구동에 그치지 않고, Github Actions를 활용한 자동화 빌드 및 정적 웹 호스팅(Github Pages)을 구축하여 배포함.

## 5. 결론

통합과학 수준의 역학 지식과 TypeScript를 결합하여 웹 기반 2차원 충돌 시뮬레이터를 성공적으로 구현함. 다만, 현재 버전은 두 가지 물리적 타협점을 가지고 있음.

1. 물체 간 충돌 시, 물체1이 강제로 정지한다고 임의로 가정함.

2. 2차원 평면에서의 충돌 법선 벡터(Normal Vector)를 정밀하게 고려하지 않아, 충격량의 작용 방향이 현실의 역학 시스템과 다소 차이가 있음.

이를 기술적으로 극복하기 위해, 향후 물리학 과정과 역학과 에너지 과정에서 운동량 보존 법칙과 2차원 벡터 연산을 추가로 심도 있게 학습하여, 어떠한 충돌 각도와 질량 비율에서도 완벽한 운동을 계산하는 강체 물리 엔진으로 본 프로젝트를 고도화할 계획임.
